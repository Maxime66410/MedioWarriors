What is the Ultimate LOD System
Ultimate LOD System is a complete system for managing and creating LODs for your meshes. With ULOD you can automatically generate LOD levels for a mesh hierarchy in your scene, directly from the Editor! After automatically generating all LODs, ULOD will still manage the LOD levels, changing the meshes according to the distance from the camera that is currently displaying an image to the user. All of this automatically. You can customize the amount of LODs that will be generated , the distance that each LOD level should be shown, the percentage of vertices that the meshes will have at each LOD level, etc. ULOD works with static meshes (Mesh Renderers) or animated meshes (Skinned Mesh Renderers).
First steps to use
Before using ULOD, your meshes must meet some requirements...

Step 1 - All of your meshes (Mesh Filters and Skinned Mesh Renderers) cannot have empty or null meshes.

The mesh filter above, has a null mesh, which will cause it to be ignored during the scanning of the ULOD component, to generate the LODs.
Step 2 - Place the "MainCamera" tag on all the main cameras in your game, avoid placing this tag on cameras that will not cover the entire screen (such as mirror cameras, mini maps, etc.).

This is particularly useful if you are going to use the "MainCamera" camera detection method in the "Ultimate Level Of Detail" component.
How to Generate LODs: Adding and configuring the ULOD component
To start everything, we must first add the component called "Ultimate Level Of Detail" inside a GameObject. If you want to create LODs for an entire Mesh hierarchy, just add ULOD to the parent GameObject for the entire hierarchy. If you want to generate LODs for a single individual mesh, add the ULOD to the GameObject that contains the MeshFilter or Skinned Mesh Renderer for that individual mesh.

Meshed GameObjects hierarchy - By adding ULOD to a GameObject parent of a GameObjects hierarchy with meshes, ULOD will be able to identify all child meshes and recognize them to generate LODs and manage the LODs of all of them.

Adding ULOD to a GameObject parent of a GameObjects hierarchy with meshes
To a single mesh - If you want to create LODs from a single mesh, just add the ULOD to the GameObject that contains this mesh.

Adding ULOD to a single mesh
Bonus - If you have a mesh hierarchy in GameObjects and a ULOD in the parent GameObject of that hierarchy, but you need to customize the LOD parameters of a mesh individually, you just need to add a ULOD to the GameObject that contains the mesh you want to customize the LOD parameters.

Adding ULOD to a child GameObject that contains a mesh
When adding a ULOD to a child GameObject, it will be possible to customize its parameters, and the parameters you define will apply only to that child GameObject. All other child GameObjects of the parent ULOD, which do not contain ULODs, will receive LOD parameters from the parent ULOD.

This child ULOD will be identified by the parent ULOD
You can see that the child ULOD has been identified by the parent ULOD, just by clicking on the parent ULOD.

Representation of parent ULOD
The ULOD child has been identified by the parent ULOD, so the scan of its parent ULOD and child ULODs will be synchronized. As soon as you start scanning on your parent object, all child ULODs will also start scanning automatically.
How to Generate LODs: Generating LODs
Now that ULOD has been added to your scene, let's do the basic settings and generate all the LODs. ULOD works as follows: After configuring all LOD creation parameters, ULOD will scan GameObjects according to the configuration you define, to find all the meshes that will have LODs created. After scanning, identifying and collecting data for each mesh, it saves all of that cataloged data and then begins creating LOD levels for each mesh. Then these levels of LODs are saved in your project (if you prefer), after that, the LODs are ready. At this point, ULOD will change meshes and LOD levels as the camera rendering them moves away or gets closer. If you need to, you can undo the creation of the LODs and then ULOD will restore all the original meshes to their proper places, erase the LOD levels previously created in your project and erase all saved data that was collected from the scanned meshes.

Now, let's go to the configuration and creation of the LODs. See the step by step below to learn how to generate LODs and make ULOD work.

Select the mesh scanning mode - Before generating LODs, ULOD needs you to show it how you want the scan to be done. If you added your ULOD to a GameObject parent of a mesh hierarchy, select "Scan In Children GameObjects Only". If you added your ULOD to a single mesh and you do not want ULOD to scan for child GameObjects for mesh, simply select "Scan In This GameObject only".

In the example, the option to scan for meshes on child GameObjects only was selected.
Select the quantity of LODs you want to create, and the quality of each LOD - Select the number of LODs you want to create first. You can customize the quality level that each mesh will have at each LOD level. The lower the "percentage of vertices" the lower the quality of that LOD, the higher the "percentage of vertices", the higher the quality of that LOD.

LOD levels that will be automatically generated for each scanned mesh
Select the distance at which each LOD level will appear - Finally, we need to set the minimum distance required for each LOD level to appear. The lower the quality of an LOD level, the greater the minimum distance required for it to appear.

Here, you can also define the minimum distance required for culling to occur in the meshes. When culling occurs, all meshes will be deactivated and will no longer be rendered until the camera approaches closer than the distance required for culling, then the meshes will be visible again.

In this example we have 4 levels of LOD in total.
LOD_0 (it is the original mesh of your model, with all vertices).
And then there are the LODs LOD_1, LOD_2 and LOD_3. The higher the level, the lower the mesh quality must be. Therefore, the higher the level, the greater the minimum distance required for that level to be seen by the player.

As LOD_0 is the original mesh, it is not possible to customize the minimum distance to see it. The minimum distance to view the LOD_0 is 0 units (Unity's unit of measurement, commonly associated with meters).Then the minimum distance to see LOD_1 is 8 Units, the minimum distance to see LOD_2 is 20 Units and the minimum distance to see LOD_3 is 50 Units. As we leave the Culling option enabled, we can select the minimum distance for culling to occur (we select 60 Units), if the camera moves away from a distance greater than 60 Units, the meshes have been culled and will no longer be visible until the camera stay at a distance of less than 60 Units. If we choose to disable Culling, and the camera moves away from a distance greater than 50, the visible LOD will always be LOD_3, culling will not occur because it is disabled.
Finally, generating the LODs - Now, just click on the "Scan All Meshes And Generate LODs" button. After that, ULOD will scan all meshes it finds and create LOD levels for each one automatically, and after that your LODs will be ready! The ULOD component will automatically change LOD levels according to the current distance from the camera rendering the meshes. This camera is automatically identified by ULOD.

Incredible Tips That Can Be Useful: Deactivating the LOD changes and displaying only original meshes temporarily at runtime
In some situations, we need to disable LODs and display only the original quality meshes, no matter how far these meshes are from the camera. We have as an example, a shooting game, where there are weapons with a system of sight with zoom.

Let's say the player picks up his weapon, looks at the horizon, switch to his aim and zoom out. We would not like to display a lot of low quality meshes (LOD_3) to the player, correct?

Player using sight and seeing distant models, however, in original quality
For this, ULOD has a special static C# API, with the function of disabling LOD management of all ULOD components in your scene. Thus, while the player is looking in the crosshairs, you can keep LOD management off, and when the player returns to normal view, you can activate LOD management again and all ULOD components will switch back to LOD meshes according to the distance from the current camera.
Script C# 


using UnityEngine;
using MTAssets.UltimateLODSystem;
                        
public class Test : MonoBehaviour {
                        
    void Start () {
        //Disable all ULOD components LODs management and restore original meshes for all ULODs
        UltimateLevelOfDetailGlobal.EnableGlobalULodSystem(false);
                
        //Enable all ULOD components LODs management
        UltimateLevelOfDetailGlobal.EnableGlobalULodSystem(true);
                
        //is Global ULOD components management current enabled?
        bool isEnabled = UltimateLevelOfDetailGlobal.isGlobalULodSystemEnabled();
    }
}

If you delete the scan and LOD meshes generated by ULOD, these components will be removed automatically as well.

Incredible Tips That Can Be Useful: Optimization tips for using ULOD
ULOD has been tested and is a package focused on optimizing your project. ULOD has been tested with thousands of GameObjects in one scene, and has been able to transition LODs smoothly, without causing bottlenecks or performance drops, however, some practices must be taken into account to keep your game even more optimized and even more functional and intuitive to perform maintenance in the future. Some of them are...

Avoid many unnecessary GameObjects - A golden tip is, the less active GameObjects your scene has, the more optimized it will be. So, the tip here is, avoid your camera seeing MANY GameObjects on the screen, and whenever possible combine your meshes of your scene or characters in addition to reducing the count of GameObjects in the scene, also reducing the DrawCalls generated by each mesh in each GameObject different. You can use Easy Mesh Combiner MT to combine the static meshes of your scene (Mesh Filters and Mesh Renderers) into just one mesh. And you can use the Skinned Mesh Combiner MT to combine all the meshes of your animated characters into one mesh. This can really bring huge performance gains, and after combining your meshes, you will only have 1 mesh, so just use ULOD to create LODs from that mesh. You will experience an explosion of performance in your game after doing this!

Avoid adding ULOD to each of your meshes - As previously stated, ULOD was created with a focus on performance and is optimized for hundreds of GameObjects, however, as a complement to the previous topic, it is recommended that you avoid adding MANY ULOD components to your scene, more precisely, avoid placing a ULOD for each mesh in your scene. The most recommended is that you reduce the amount of GameObjects in your scene as much as possible, combine your meshes and add ULOD to a group of Meshes, so that a single ULOD component will manage the LODs of several meshes that are nearby and this will make things faster and allow you to invest in adding even more meshes to your scene. You can place as many ULODs as you want in your scene, but the tip here is, avoid placing unnecessary ULODs, so that you can add even more elements to your scene. For example, if you have a set of 10 trees that are close together, instead of adding a ULOD for each tree, group all these trees into a single GameObject (and if possible combine their meshes into just one) and place the ULOD in the GameObject parent of those meshes. So you will have a single ULOD managing the LOD of these 10 trees!
Incredible Tips That Can Be Useful: How to generate simplified versions of a mesh, without using the "Ultimate Level Of Detail" component and save as file
Sometimes we just need to generate simplified versions (with less vertices) of a mesh, but we don't necessarily need an LOD controller for this. With this need in mind, the "Mesh Simplifier" window was created and introduced in the Ultimate LOD System. You can now generate simplified versions of a mesh and save it in the same directory to which the original mesh is located, automatically and in an organized manner.

Representation
First, you just need to open the simplification window. It can be opened by right clicking on a GameObject in your scene and then "Simplify Mesh", or by right clicking on a model file and then "Open With Mesh Simplifier" or simply go to "Tools > Ultimate LOD System > Mesh Simplifier Tool".

To use, you simply need to provide a GameObject that contains a mesh, or a mesh directly, so choose the simplification parameters and click on "Generate Simplified Meshes". The simplified meshes will be saved in the same directory where the original mesh file is located.
Current Limitations of the ULOD
ULOD works well, however, like any tool it has its limitations. See below...

Meshes with vertices occupied the same space - Vertices that occupy the same space (or a position very very very close to another vertically atomically speaking) can be a problem for the internal mesh simplification algorithm. That said, meshes that have this characteristic can present hole problems in their LODs. ULOD will try its best to simplify these loops without any problems, but this is something that can happen.
Settings e Options Of ULOD
In the table below, you can see all the parameters and configurations of the ULOD component.

Name Description
Mode of Meshes Scanning How ULOD will use it to scan and find loops to generate LODs.
Scan Inactive Too If you choose to scan for meshes in child GameObjects, enable this and ULOD will scan even meshes that are in disabled GameObjects.
GameObjects to Ignore During Scan You can register GameObjects so that they are ignored during scanning, so even if they have meshes, meshes are not included in the processing of LODs.
Save Generated LODs In Assets If you activate this option, ULOD will save all levels of LODs generated from their original fabrics, in your project files. Note that: If you are going to create prefabs, it is recommended that you leave this option enabled.
Levels Of Details To Generate Here you define the amount of LODs you want to generate.
Percent Of Vertices In LOD X The quality level of each the LOD_X mesh.
Skinned Anim Compat Mode In rare cases, some models may experience animation problems in the LOD loops. To prevent this problem, enable this option. This option does not require or reduce your game's performance, however, if you notice that some animation of your character is behaving unexpectedly in LOD loops, try to disable this.
Prevent Artifacts Or Deform If this option is enabled, ULOD will use an internal algorithm to prevent its LODs from becoming too deformed or showing distortions or artifacts. The purpose of this function is to prevent the reduction of vertices from spoiling the shape of your model. This can make the rate of reduction of vertices lower, in other words, it sacrifices the reduction of vertices due to the maintenance of the good shape of your LOD.
Force Of Simplification The recommended simplification force is "Normal". However, if you have a mesh with thousands of vertices, the Normal force may not be enough to reduce many vertices in that mesh. The "Normal" force works very well with the "Prevent Artifacts Or Deform" option and can reduce many vertices, however, if you need to reduce even more vertices, it may be necessary to increase the simplification force (and of course, reduce the percentage of vertices at each LOD level). Forces greater than "Normal" do not work with the "Prevent Artifacts Or Deform" option enabled.
Culling Mode Here you choose the Culling method that will be performed when the camera has a certain distance defined by you. The most recommended is "CullingMeshes".
Min Distance To View LOD X In these options, you can customize the minimum distance required for each LOD level to be seen by the camera that is rendered in the mesh. The higher the LOD level (the lower the mesh quality), the greater the minimum distance must be for that level to be seen.
Min Distance Of View For Cull Here you define the minimum distance for Culling to occur in your meshes.
Draw Gizmo On This Pivot To calculate the distance between the mesh and the camera rendering the mesh, ULOD uses as point A, the pivot of the GameObject that ULOD is in and as point B, the pivot of the camera that is rendering the GameObject ULOD. If this option is activated, the ULOD will show with a sphere, the position where the pivot of this ULOD is. (Works only in the Editor).
Editor Skinned LODs Change This option forces the Editor to exchange LODs on Skinned fabrics. This can cause crashes in some versions of Unity Editor. Even if this option is enabled or disabled, exchanges of LODs in Skinned fabrics will normally occur in the final game built.
Runtime API of Component "Ultimate Level Of Detail": Get the LOD level that is currently on display
To see the level of LOD that a ULOD component is currently displaying, at run time, just use the API below.

Script C#


using UnityEngine;
using MTAssets.UltimateLODSystem;

public class Test : MonoBehaviour {

    //Some ULOD component
    public UltimateLevelOfDetail ulod;

    void Start () {
        //Get current LOD level of component
        int currentLodLevel = ulod.GetCurrentLodLevel();
    }
}
Runtime API of Component "Ultimate Level Of Detail": Get the current distance between the ULOD and the main camera currently in use
To find out the distance between a ULOD component and the camera that is rendering it, use the method below. Note that this method is affected by the ULOD global distance multiplier. This method returns a value multiplied by the ULOD global distance multiplier. If you want a real value, use the "GetCurrentRealCameraDistance()" method for that.

Script C#


using UnityEngine;
using MTAssets.UltimateLODSystem;

public class Test : MonoBehaviour {

    //Some ULOD component
    public UltimateLevelOfDetail ulod;

    void Start () {
        //Get current distance between ulod component (Point A) and camera (Point B)
        float distanceBetweenUlodAndCamera = ulod.GetCurrentCameraDistance();
    }
}

Runtime API of Component "Ultimate Level Of Detail": Force a ULOD component to update its LOD and rendering calculations
ULOD updates the rendering of LODs and only changes LODs when it is really necessary, that is, if the distance from the camera changes, if the LOD has changed, etc. ULOD does several checks to ensure that it will not make unnecessary LOD changes while consuming performance for no reason. But there are times when we need to show changes more precisely, such as when you change a mesh in the "Ultimate Level Of Detail Meshes" component. In doing so, the new mesh you provided in that component will only be used the next time ULOD changes LODs, so use the method below to force ULOD to update the display of LODs and rendering calculations.

Script C#


using UnityEngine;
using MTAssets.UltimateLODSystem;
                                                                
public class Test : MonoBehaviour {
                                                                
    //Some ULOD component
    public UltimateLevelOfDetail ulod;
                                                                
    void Start () {
        //Disable the ULOD
        ulod.ForceThisComponentToUpdateLodsRender();
    }
}
Runtime API of Component "Ultimate Level Of Detail": Disable or enable all ULODs in the scene, globally
As quoted here (please read this), you can disable or enable LODs management for all ULOD components in your scene, just with a global parameter. When disabling this management, all ULODs in your scene will display only the original meshes, when reactivating this management, all ULODs in your scene will again display the LODs currently corresponding to the distance between them and the camera that renders them.
Runtime API of Component "Ultimate Level Of Detail": Changing the global LOD distance multiplier
The Ultimate LOD System allows you to define a global distance multiplier (which automatically applies to all Ultimate LOD System components in all scenes). This multiplier affects the distances between the LODs of the ULOD components. The default value is "1.0", that is, if you have defined that the minimum distance to render LOD_1 is 20 Units in a ULOD component, then that LOD_1 will only be rendered when the camera is 20 Units away from the ULOD component .

If you change this multiplier to "2.0", then LOD_1 will be rendered with 10 Units, if you change to "0.0" then LODs will never be rendered and the original meshes will always be displayed. This distance multiplier also applies to Culling distances.

This can be relatively useful if you want to put a LOD quality setting in your game, where strong machines don't need the LOD system to work the same way as they do on a cell phone for example.

Script C#


using UnityEngine;
using MTAssets.UltimateLODSystem;
                
    public class Test : MonoBehaviour {
                
        void Start () {
            //Define a new multiplier
            UltimateLevelOfDetailGlobal.SetGlobalLodDistanceMultiplier(1.5f);
            
            //Get current multiplier
            float multiplier = UltimateLevelOfDetailGlobal.GetGlobalLodDistanceMultiplier();
        }
    }

Runtime API of Component "Ultimate Level Of Detail Meshes": Provide a new custom mesh for a LOD
The Ultimate LOD System allows you to provide your own meshes for use in LODs. To do this at runtime, just use the method below. Your mesh will be used in the LOD you want. below.

You can only provide meshes for levels 0 through 8 of LODs.

Script C#


using UnityEngine;
using MTAssets.UltimateLODSystem;
                                                                
public class Test : MonoBehaviour {
                                                                
    //Some ULOD Meshes component
    public UltimateLevelOfDetailMeshes ulodMeshes;
    //Some mesh
    public Mesh aMesh;
                                                                
    void Start () {
        //Set a new mesh to be used in level 5 of LOD in this Ultimate Level Of Detail Meshes
        ulodMeshes.SetMeshOfThisLodGroup(5, aMesh);
    }
}

Problems Solution: Holes in simplified meshes
The holes in the meshes may appear due to vertices that are occupying the same place as other vertices in your mesh, but if this does not happen to your mesh, try to activate the option "Prevent Artifacts Or Deform" before generating your LODs.
Problems Solution: Skinned Mesh Renderers animations not working
If this happens, it may be due to some problem with the binding of your bones. This is usually fixed by enabling the "Skinned Anims Compat Mode" option.
Problems Solution: Objects are not animated correctly after generating LODs
The most likely cause for this is that you have objects that are parents under bones that move with the animations. There is currently no way to deal with this limitation, and the best way to do this is to use lod groups nested with a ULOD component. Any object you know to be a parent under a bone must have its own LOD Group with a ULOD.
Problems Solution: The simulation of LOD changes in Skinned Mesh Renderers does not occur in my Unity editor
The Ultimate LOD System is fully compatible with Skinned meshes. ULOD will scan and change LODs of Skinned meshes without any problems, however, LOD changes simulation in the Unity Editor is disabled by default. This is because some versions of Unity have limitations that cause Editor to crash if skinned mesh LODs are changed.

Even if the LOD changes simulation is disabled in the Editor, if you click on the ULOD component that manages the LODs of the Skinned Mesh Renderer meshes, you will see that it shows the LOD that is currently being rendered. So you have a sense of distance for each LOD.

It is important to note that even if the LOD changes simulation is disabled by default in the Editor, if you compile and create your game (regardless of platform) the LOD changes will work without problems, and the meshes will be changed for LODs according to the distance from the camera. The non-functioning of the LOD changes simulation only applies to the Editor.

Now that the important thing has been explained, it is also important to know, that there is an option to force the Ultimate LOD System to run the LOD changes simulation in the Editor. Just activate the "Editor Skinned LODs Change" option in the "Ultimate Level Of Detail" component and the Ultimate LOD System will do the LOD changes simulations in the Editor as well. But remember that this can make some versions of Unity crash.

Problems Solution: Flat things like walls, planes and squares with few vertices seem to warp as they turn into LODs
The Ultimate LOD System will always try to maintain the original shape of the meshes, be it a character, car, etc, but this can happen with thin walls (in walls or parts of houses that are composed of few vertices). The vertex reduction algorithm of the Ultimate LOD System will always try to render the vertices as much as possible, so when flat meshes that have few vertices pass through the algorithm, they have the vertices reduced from being square to being triangles for example, if a wall very thin composed of 4 vertices only, passes through the algorithm, the algorithm will leave the wall with 3 vertices, transforming it into a triangle and so on.

Representation. After configuring ULOD to ignore wall meshes, the problem will not occur again. As the wall meshes have few vertices, they will have a negligible impact on the game's performance, even if they do not have generated LODs.
The best way to do this is to configure the "Ultimate Level Of Detail" component to ignore parts such as houses, or the walls of houses, to avoid deformations in these areas, so ULOD will generate LODs without deforming important parts of a construction.
